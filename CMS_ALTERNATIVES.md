# CMS Alternatives for Hugo Site

Since you want a different solution than Decap CMS, here are alternatives:

## Option 1: TinaCMS (formerly Forestry) ‚≠ê

**Pros:**
- Git-based (commits to GitHub)
- Visual editor
- Works with Hugo
- Free for personal use

**Setup:**
- Sign up at tina.io
- Connect your GitHub repo
- Configure content types
- Edit through TinaCMS interface

**Cost:** Free for personal projects

## Option 2: GitHub Web Editor (Simplest)

**Pros:**
- No setup needed
- Direct GitHub integration
- Free
- You already have access

**How it works:**
- Edit files directly on GitHub
- Changes auto-deploy via GitHub Actions
- Use GitHub's markdown editor

**No CMS needed** - just edit files on GitHub!

## Option 3: Prose.io

**Pros:**
- Simple markdown editor
- Connects to GitHub
- Free and open source

**Setup:**
- Visit prose.io
- Connect GitHub repo
- Edit markdown files visually

## Option 4: Stackbit

**Pros:**
- Visual page builder
- Works with Hugo
- Git-based

**Cons:**
- More complex setup
- May have costs for advanced features

## Option 5: Custom Simple Editor

**Pros:**
- Full control
- Lightweight
- Can be tailored to your needs

**Cons:**
- Requires development
- More maintenance

## Option 6: Headless CMS (Strapi, Contentful, etc.)

**Pros:**
- Full-featured CMS
- API-based
- Can integrate with Hugo

**Cons:**
- Requires backend/server
- More complex setup
- May have costs
- Need to sync content to GitHub

## Recommendation

**For simplicity:** Use **GitHub Web Editor** - no setup, works immediately, free.

**For visual editing:** Try **TinaCMS** - similar to Decap but different interface.

Which approach interests you? Or do you have something else in mind?
